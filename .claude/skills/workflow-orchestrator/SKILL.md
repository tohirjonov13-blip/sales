# Workflow Orchestrator

Единый набор предпочтений пользователя, перенесённый из всех проектов в один пакет.
Эти правила применяются всегда, в любом репозитории, куда положен `.claude/`.

## 1. Backup перед глобальными изменениями (обязательно)

Перед тем как менять **архитектуру, логику или дизайн** крупным блоком
(переписать секцию, рефакторинг модуля, замена структуры, редизайн вёрстки) —
сначала сделай резервную копию изменяемого файла:

```bash
.claude/hooks/backup-block.sh <файл> <короткая-метка>
# пример:
.claude/hooks/backup-block.sh index.html redesign-header
```

Копия ложится в `.claude/backups/<timestamp>-<метка>/` с сохранением пути.
Это страховка для отката, не замена git. Мелкие правки (опечатка, одна строка,
переименование переменной) бэкапить не нужно — только значимые блоки.

## 2. Keep-awake для долгих задач

Длинные процессы (сборки, миграции, тесты, пакетная обработка) запускай через
обёртку, чтобы машина не заснула и не оборвала задачу:

```bash
.claude/hooks/keep-awake.sh <команда...>
# пример:
.claude/hooks/keep-awake.sh npm run build
```

- На **macOS** это вызывает настоящий `caffeinate -dimsu`.
- В **облачном/Linux** окружении раннер не засыпает — обёртка просто запускает команду.

## 3. Авто-апрув команд (кроме удаления файлов)

Настроено в `.claude/settings.json`:
- большинство команд и правок файлов выполняются **без переспроса**;
- команды **удаления** (`rm`, `rmdir`, `unlink`, `shred`, `trash`, `git clean`,
  `git rm`, `dd`, `sudo rm`) переведены в режим **ask** — всегда переспрашивают.

Если нужно удалить файл — делай это явной командой удаления (не пряча её внутри
`bash -c "..."` или пайпа), чтобы правило сработало и запрос подтверждения появился.

## 4. Подключение superpowers

Набор скиллов superpowers подключается как плагин (см. `.claude/README.md`,
раздел «superpowers»). Если в сессии доступны скиллы superpowers — используй их
для подходящих задач (планирование, отладка, ревью и т.п.).

## 5. Обновление памяти после задачи

После завершения значимой задачи обнови `CLAUDE.md` проекта: что изменилось
(архитектура/логика/дизайн), ключевые решения, контекст для будущих сессий.
Stop-hook (`.claude/hooks/stop-memory-reminder.sh`) один раз напомнит, если были
изменения в рабочем дереве, а `CLAUDE.md` не тронут. Если фиксировать нечего —
явно подтверди это и заверши.

## Порядок работы над задачей

1. Понял задачу → если предстоят глобальные изменения, сделай **backup** (п.1).
2. Долгие команды — через **keep-awake** (п.2).
3. Удаление файлов — только явной командой, подтверждай (п.3).
4. Используй **superpowers**, где уместно (п.4).
5. По завершении — обнови **CLAUDE.md** (п.5).
