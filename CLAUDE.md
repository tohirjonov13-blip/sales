# Project memory & workflow rules

> Единые правила работы для этого проекта. Подробности — в
> `.claude/skills/workflow-orchestrator/SKILL.md`.

## Рабочий процесс (всегда)

1. **Backup перед глобальными изменениями.** Прежде чем переписывать
   архитектуру / логику / дизайн крупным блоком, сделай копию файла:
   `.claude/hooks/backup-block.sh <файл> <метка>`. Мелкие правки — без бэкапа.
2. **Долгие задачи — через keep-awake:** `.claude/hooks/keep-awake.sh <команда>`.
3. **Удаление файлов** требует подтверждения (настроено в `.claude/settings.json`);
   удаляй только явными командами.
4. **superpowers** — используй скиллы плагина, где уместно.
5. **UI/дизайн интерфейса** — выполняй через скилл `frontend-designer`, а не вручную.
6. **Subagent-driven задачи** — последним шагом запускай агента-аудитора на модели
   `opus`, который ищет баги и недоработки остальных агентов.
7. **После задачи обнови этот файл** (память проекта): что изменилось, ключевые
   решения, контекст для будущих сессий.
